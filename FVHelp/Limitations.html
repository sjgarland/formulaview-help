The current release of Formula View has the following limitations.

Web access required

Formula View, like all add-ins for Office 365, can only be used when connected to the web.

The math view uses the web-based typesetting service provided by MathJax.org. Hence the math view is available only if that service is available.

Notations in formulas

Formula View does not recognize R1C1 notations for references.

Formula View treats identifiers such as ABCD2 and A1234567 as references to cells, rather than as user-defined names, even though they exceed Excel's limits on the maximum numbers of rows and columns in a spreadsheet.

Formula View does not recognize structured references, such as Inventory[InStock], to columns in tables.

Array expressions

Formula View may display incorrect values for expressions that occur in array formulas or in functions like SUMPRODUCT, which treat their arguments as array formulas.

Volatile functions

Some functions in Excel are volatile in that they can produce different results each time they are evaluated. For example, RND and RNDBETWEEN deliberately produce different (and unpredictable) values. The functions NOW, TODAY, OFFSET, and INDIRECT are also volatile, as are INFO, CELL, and SUMIF for certain arguments. Excel recommends using volatile functions sparingly.

Because Formula View uses Excel to compute values for display in the tree view, it can change the value of the viewed formula. Furthermore, the value it shows for a volatile function in the tree view may be inconsistent with the values it shows for elements above that function. (This is a bug that may be fixed in a future release.)

Limitations imposed by Excel for the web

Excel for the web does not allow the width of the Formula View pane to be increased. Neither does it allow the pane to be undocked.
